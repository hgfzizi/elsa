订单费用查询模块 PRD

1. 模块概述

订单费用查询模块用于管理和查询电商订单的物流费用信息，支持多维度筛选、数据导出和费用统计功能。

2. 功能需求

2.1 查询功能

  • 支持按状态码标签筛选（全部/待发货/已发货）
  • 支持按仓库代码筛选（MD001/DE001，根据账号绑定动态展示）
  • 支持按承运商筛选（FEDEX/UPS/USPS/AMAZON/GOFO/LTL/One-Touch/未知）
  • 支持按配送渠道筛选（ZYBQ_FEDEX/ZYBQ_UPS/ZYBQ_SY/ZYBQ_DS）
  • 支持按创建时间范围、重量区间、费用区间筛选
  • 支持按出库单号精确匹配（多个以空格隔开）

2.2 数据展示

  • 表格展示订单详细信息（出库单号、业务类型、承运商、平台店铺、费用明细等共22列）
  • 实时统计当前筛选条件下的订单总金额（红色高亮，右侧显示）
  • 支持分页浏览（默认每页10条）

2.3 操作功能

  • 导出：弹窗选择导出字段，默认全选，导出为Excel文件
  • 重置：清空所有筛选条件，恢复默认列表
  • 查询：应用所有筛选条件，刷新列表和总金额

3. 业务规则

  • 总成本 = 出库费 + 基础价 + 住宅附加费 + 超长费 + 超重费 + 超大件费 + 偏远 + 超偏远 + 燃油附加费
  • 状态码逻辑：待发货 = 待确认/待拣货/待打印/待配货/待复核；已发货 = 已出库
  • 仓库选项根据当前账号绑定的仓库代码动态展示
  • 数据按费用发生时间倒序排列
